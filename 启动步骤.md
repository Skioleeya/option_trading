# SPY 0DTE DashBoard v3.0 — 运营操作协议 (Operational Protocol)

> [!NOTE]
> 本协议旨在确保系统处于“零污染”环境启动。作为华尔街期权分析师，我们首先确保旧进程已出清。

---

## 🔹 Step 0: 环境清场 (Environment Sanitation)
在启动任何服务前，必须强制清理旧的 Python 进程和端口占用。
```powershell
# 终止所有 Python 和 Uvicorn 进程
Get-Process python, uvicorn -ErrorAction SilentlyContinue | Stop-Process -Force
# 清理 8001 端口占用
Get-NetTCPConnection -LocalPort 8001 -ErrorAction SilentlyContinue | ForEach-Object { Stop-Process -Id $_.OwningProcess -Force }
```

## 🔹 Step 1: 核心基础架构 (Core Infra)
启动分布式缓存层，这是所有实时希腊值 (Greeks) 计算的基础。
```powershell
# 运行 Redis Server (端口: 6380)
.\scripts\redis-start.bat
```
- **验证**: `redis-cli -p 6380 ping` 应返回 `PONG`。

## 🔹 Step 2: 智能分析引擎 (Backend - Alpha Engine)
加载 OptionChainBuilder, DepthEngine 以及 AgentG 决策逻辑。
```powershell
# 在项目根目录运行 (v3.0 架构)
$env:PYTHONPATH='.'
python -m uvicorn main:app --host 0.0.0.0 --port 8001 --log-level info
```
- **验证**: 见日志输出 `[AppContainer] All services initialized`。
- **性能**: 确认 `build_payload` 耗时 < 50ms（GPU 模式）。

## 🔹 Step 3: 视觉决策控制台 (Frontend - UI Console)
启动基于 Vite 的 TradingView 集成控制面板。
```powershell
cd frontend
npm run dev
```
- **终端地址**: [http://localhost:5173](http://localhost:5173)

---

## 📊 运行监控 (Post-Flight Checks)
1. **数据流动性**: 确认前端 ATM Decay Chart 并非水平直线。
2. **决策信号**: 后端日志应显示 `AgentG Final Signal`，即使是 `NO_TRADE` 也代表系统链路闭环。

## ⚠️ 应急处置 (Risk Management)
- **数据中断**: 检查 `Longport Token` 是否过期（`scripts/diag_token.py`）。
- **计算延迟**: 若 `build_payload` 超过 500ms，需检查 GPU 负载。
