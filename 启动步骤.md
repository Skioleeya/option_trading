# SPY 0DTE Dashboard v3.0 — 运营启动协议（建议版）

> [!IMPORTANT]
> 新流程改为 **先探测、再启动**，避免重复启动导致 `WinError 10048 (8001 端口已占用)`。
>
> 本文档显式覆盖三类常见失败源：
> 1) `pythonw.exe` / WindowsApps 桩文件导致后端无法启动；
> 2) 后端已运行却再次启动，引发端口冲突；
> 3) LongPort 启动连通性失败时，需要 strict -> degraded 明确分支。

---

## 🔹 Step 0: 先探测当前状态（不要先清场）

```powershell
# Backend
try {
    $backend = Invoke-WebRequest http://127.0.0.1:8001/health -UseBasicParsing -TimeoutSec 3
    "backend: UP ($($backend.StatusCode))"
} catch {
    "backend: DOWN"
}

# Frontend
try {
    $frontend = Invoke-WebRequest http://127.0.0.1:5173 -UseBasicParsing -TimeoutSec 3
    "frontend: UP ($($frontend.StatusCode))"
} catch {
    "frontend: DOWN"
}

# Redis
$redisUp = [bool](Get-NetTCPConnection -LocalPort 6380 -State Listen -ErrorAction SilentlyContinue)
"redis: " + ($(if($redisUp){"UP"}else{"DOWN"}))
```

结论：
- 若 `backend: UP (200)`，后端已就绪，**不要重复启动**。
- 若 `frontend: UP (200)`，前端已就绪，**不要重复启动**。
- 仅对 DOWN 的组件执行后续启动步骤。

## 🔹 Step 1: 按需清场（仅在需要重启时）

```powershell
# 仅在你确认要重启整个链路时执行
Stop-Process -Name python,uvicorn,redis-server,node -ErrorAction SilentlyContinue -Force
```

---

## 🔹 Step 2: 启动 Redis（推荐单独窗口前台运行）

```powershell
Set-Location E:\US.market\Option_v3
$redisExe  = (Resolve-Path .\infra\bin\redis-server.exe).Path
$redisConf = (Resolve-Path .\infra\redis\redis.conf.local).Path
& $redisExe $redisConf
```

验证（无需 `redis-cli`）：
- `Get-NetTCPConnection -LocalPort 6380 -State Listen`
- 或 `Test-NetConnection 127.0.0.1 -Port 6380`

---

## 🔹 Step 3: 启动后端（先 strict，失败再 degraded）

### 3.1 strict（默认）

```powershell
Set-Location E:\US.market\Option_v3
$env:PYTHONPATH='.'
Remove-Item Env:LONGPORT_STARTUP_STRICT_CONNECTIVITY -ErrorAction SilentlyContinue
Remove-Item Env:LONGBRIDGE_STARTUP_STRICT_CONNECTIVITY -ErrorAction SilentlyContinue
python -m uvicorn main:app --host 0.0.0.0 --port 8001
```

### 3.2 若 strict 因 LongPort 连通性失败，切换 degraded

```powershell
Set-Location E:\US.market\Option_v3
$env:PYTHONPATH='.'
$env:LONGPORT_STARTUP_STRICT_CONNECTIVITY='false'
$env:LONGBRIDGE_STARTUP_STRICT_CONNECTIVITY='false'
python -m uvicorn main:app --host 0.0.0.0 --port 8001
```

后端验证：
- `Invoke-WebRequest http://127.0.0.1:8001/health -UseBasicParsing`

若出现 `WinError 10048`：
1) 先执行健康检查；若返回 200，表示已有实例在跑，无需处理。  
2) 需要替换实例时再释放端口：

```powershell
$pid8001 = (Get-NetTCPConnection -LocalPort 8001 -State Listen | Select-Object -First 1).OwningProcess
Get-Process -Id $pid8001
Stop-Process -Id $pid8001 -Force
```

## 🔹 Step 4: 启动前端

```powershell
Set-Location E:\US.market\Option_v3\l4_ui
npm run dev -- --host 0.0.0.0 --port 5173
```

前端验证：
- `Invoke-WebRequest http://127.0.0.1:5173 -UseBasicParsing`

## 📊 启动后检查（最小闭环）

1. 后端：`/health` 返回 200 且 `status=ok`
2. 前端：`5173` 返回 200
3. Redis：`6380` 处于 LISTEN
4. UI：Header WebSocket 绿色连接状态，ATM Decay 非水平直线

## ⚠️ 应急处置

| 问题 | 处置 |
|------|------|
| 端口 8001 冲突 | 先 `Invoke-WebRequest /health` 判断是否已运行；仅在需要替换时 kill 端口占用进程 |
| 数据中断 | `python scripts/diag_token.py` 检查 LongPort Token |
| 全链路复位 | `taskkill /F /IM python.exe /IM pythonw.exe /IM redis-server.exe /IM node.exe` |
