# SPY 0DTE DashBoard v3.0 — 运营启动协议

> [!IMPORTANT]
> **Windows 10 优化版**：本协议已针对静默运行进行了优化，避免弹窗干扰。
>
> 本文档已修正两类常见失败源：
> 1) `pythonw.exe` 在部分系统会命中 WindowsApps 桩文件，导致后端无法启动；
> 2) 本地未安装 `redis-cli` 时，原验证命令不可用。

---

## 🔹 Step 0: 环境清场 (静默)

```powershell
# 终止所有 Python / Uvicorn / Redis 进程
Stop-Process -Name python, uvicorn, redis-server -ErrorAction SilentlyContinue -Force
```

---

## 🔹 Step 1: Redis 缓存层 (后台，推荐显式启动)

```powershell
$redisBin = ".\infra\bin\redis-server.exe"
$redisConf = ".\infra\redis\redis.conf.local"

if (Test-Path $redisBin) {
    # 显式启动 Redis（静默）
    Start-Process -FilePath $redisBin -ArgumentList $redisConf -WindowStyle Hidden -WorkingDirectory (Resolve-Path ".\infra\redis")
} else {
    Write-Warning "未找到 $redisBin。继续启动后端；后端会尝试自动拉起 Redis。"
}
```

- **验证（无需 redis-cli）**:
  - `netstat -ano | findstr :6380` 出现 `LISTENING`
  - 或 `Test-NetConnection 127.0.0.1 -Port 6380`

---

## 🔹 Step 2: 后端分析引擎 (静默)

```powershell
# 使用 python 可执行文件（避免 pythonw WindowsApps 桩文件问题）
$env:PYTHONPATH='.'
$py = (Get-Command python).Source
Start-Process -FilePath $py -ArgumentList "-m uvicorn main:app --host 0.0.0.0 --port 8001" -WindowStyle Hidden -WorkingDirectory (Get-Location)
```

- **验证**: `http://localhost:8001/health` 返回 `{"status":"ok"}`
- **日志**: 若需查看日志，运行 `Get-Content .\logs\backend.log -Wait` (若已配置日志文件)

---

## 🔹 Step 3: 前端控制台

```powershell
# 前端通常需要在集成终端保持可见以查看编译状态
cd l4_ui
npm run dev
```

- **地址**: [http://localhost:5173](http://localhost:5173)

---

## 📊 启动后检查

1. **ATM Decay Chart**：确认图表并非水平直线（数据链路正常）。
2. **后台进程**: `Get-Process | Where-Object {$_.Name -match 'python|redis'}` 确认处于运行状态。
3. **WebSocket**：前端 Header 显示绿色连接状态圆点。

---

## ⚠️ 应急处置

| 问题 | 处置 |
|------|------|
| 端口 8001 被占用 | `Stop-Process -Id (Get-NetTCPConnection -LocalPort 8001).OwningProcess -Force` |
| 数据中断 | 检查 Longport Token：`python scripts/diag_token.py` |
| 无法复位 | 运行 `taskkill /F /IM python.exe /IM pythonw.exe /IM redis-server.exe /IM node.exe` |
