# SPY 0DTE DashBoard v3.0 — 运营启动协议

> [!NOTE]
> 在启动任何服务前，必须确保旧进程已完全出清（零污染启动）。

---

## 🔹 Step 0: 环境清场

```powershell
# 终止所有 Python / Uvicorn 进程
Get-Process python, uvicorn -ErrorAction SilentlyContinue | Stop-Process -Force

# 清理 8001 端口残留进程（防止端口占用）
$pid8001 = (netstat -ano | findstr :8001 | findstr LISTENING) -replace '\s+',' ' | ForEach-Object { ($_ -split ' ')[-1] }
if ($pid8001) { taskkill /F /PID $pid8001 }
```

---

## 🔹 Step 1: Redis 缓存层

```powershell
.\scripts\redis-start.bat
```

- **验证**: `redis-cli -p 6380 ping` → 应返回 `PONG`

---

## 🔹 Step 2: 后端分析引擎

```powershell
# 在项目根目录运行
$env:PYTHONPATH='.'
python -m uvicorn main:app --host 0.0.0.0 --port 8001 --log-level info
```

- **验证**: 日志出现 `[Lifespan] All services and background loops initialized`
- **性能**: 确认 `build_payload` 耗时 < 50ms（GPU 模式）
- **端口**: `http://localhost:8001/health` 返回 `{"status":"ok"}`

> **注**：v3.1 重构后，AppContainer 已拆解为 `app/lifespan.py`（初始化编排）、`app/loops/`（异步循环）、`app/routes/`（路由）。

---

## 🔹 Step 3: 前端控制台

```powershell
cd l4_ui
npm run dev
```

- **地址**: [http://localhost:5173](http://localhost:5173)

---

## 📊 启动后检查

1. **ATM Decay Chart**：确认图表并非水平直线（数据链路正常）。
2. **后端日志**：确认 `[L1ComputeReactor] compute` 日志每秒 1 次（计算循环正常）。
3. **WebSocket**：前端 Header 显示绿色连接状态圆点。

---

## ⚠️ 应急处置

| 问题 | 处置 |
|------|------|
| 端口 8001 被占用 | `netstat -ano \| findstr :8001`，`taskkill /F /PID <PID>` |
| 数据中断 | 检查 Longport Token：`python scripts/diag_token.py` |
| 计算延迟 > 500ms | 检查 GPU 负载；或强制降至 Numba Tier |
| WS 一直重连 | 检查后端 Redis 是否启动：`redis-cli -p 6380 ping` |
