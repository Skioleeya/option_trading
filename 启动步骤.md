# Option_v3 启动步骤（实测可用版）

本文档按 2026-03-11 实测成功流程编写，目标是一次性拉起：
- Redis（6380）
- 后端 FastAPI/Uvicorn（8001）
- 前端 Vite（5173）

---

## 1. 先探测（probe-first）

在 `PowerShell` 执行：

```powershell
Set-Location E:\US.market\Option_v3

try { (Invoke-WebRequest http://127.0.0.1:8001/health -UseBasicParsing -TimeoutSec 3).StatusCode } catch { "BACKEND_DOWN" }
try { (Invoke-WebRequest http://127.0.0.1:5173 -UseBasicParsing -TimeoutSec 3).StatusCode } catch { "FRONTEND_DOWN" }
cmd /c netstat -ano | findstr ":6380 .*LISTENING"
```

判定：
- `/health = 200` 说明后端已运行，勿重复启动。
- `5173 = 200` 说明前端已运行，勿重复启动。
- `6380 LISTENING` 说明 Redis 已运行。

---

## 2. 按需启动 Redis

> 仅当 6380 未监听时执行。

```powershell
Set-Location E:\US.market\Option_v3
$redisBin  = "E:\US.market\Option_v3\infra\bin\redis-server.exe"
$redisConf = "E:\US.market\Option_v3\infra\redis\redis.conf.local"

Start-Process -FilePath $redisBin -ArgumentList "`"$redisConf`"" -WorkingDirectory "E:\US.market\Option_v3"
```

验证：

```powershell
cmd /c netstat -ano | findstr ":6380 .*LISTENING"
```

---

## 3. 启动后端（先 strict，失败回退 degraded）

### 3.1 strict 启动（默认）

```powershell
Set-Location E:\US.market\Option_v3
Start-Process -FilePath "cmd.exe" -ArgumentList "/c","set PYTHONPATH=.&& python -m uvicorn main:app --host 0.0.0.0 --port 8001" -WorkingDirectory "E:\US.market\Option_v3"
Start-Sleep -Seconds 8
try { (Invoke-WebRequest http://127.0.0.1:8001/health -UseBasicParsing -TimeoutSec 4).StatusCode } catch { "BACKEND_DOWN" }
```

### 3.2 strict 失败时回退 degraded

如果上一步不是 `200`，执行：

```powershell
Set-Location E:\US.market\Option_v3
Start-Process -FilePath "cmd.exe" -ArgumentList "/c","set PYTHONPATH=.&& set LONGPORT_STARTUP_STRICT_CONNECTIVITY=false&& set LONGBRIDGE_STARTUP_STRICT_CONNECTIVITY=false&& python -m uvicorn main:app --host 0.0.0.0 --port 8001" -WorkingDirectory "E:\US.market\Option_v3"
Start-Sleep -Seconds 12
try { (Invoke-WebRequest http://127.0.0.1:8001/health -UseBasicParsing -TimeoutSec 4).StatusCode } catch { "BACKEND_DOWN" }
```

## 5. 一次性验活（最终检查）

```powershell
try { (Invoke-WebRequest http://127.0.0.1:8001/health -UseBasicParsing -TimeoutSec 3).StatusCode } catch { "BACKEND_DOWN" }
try { (Invoke-WebRequest http://127.0.0.1:5173 -UseBasicParsing -TimeoutSec 3).StatusCode } catch { "FRONTEND_DOWN" }
cmd /c netstat -ano | findstr ":6380 .*LISTENING"
```

成功标准：
- 后端：`200`
- 前端：`200`
- Redis：`6380 LISTENING`

---

## 6. 常见问题

- 后端 strict 起不来：直接按第 3.2 节 degraded 启动。
- 端口冲突：先 probe，确认是否已有健康实例，避免重复起服务。
- 前端首次慢启动：延长 `Start-Sleep` 到 15~20 秒再验活。
