## 🚀 本地Windows启动方式

### 方法一：常驻前台自动启动器（推荐） ⭐⭐⭐

**文件**: `start_system.bat`

**操作**:
1. 双击 `start_system.bat`
2. 启动器前台运行，显示实时状态
3. 交易日开盘前3分钟自动启动子系统
4. 收盘后2分钟自动关闭所有终端窗口
5. 按 Ctrl+C 手动关闭启动器

**优点**:
- ✅ **可视化管理**: 实时显示状态和倒计时
- ✅ **全自动**: 根据交易日历智能调度
- ✅ **终端清理**: 自动关闭所有黑色终端窗口
- ✅ **手动控制**: 支持 Ctrl+C 优雅关闭
- ✅ **用户友好**: 清楚的状态反馈

**启动器界面示例**:
```
======================================================================
🏠 美股广度监控系统 - 常驻前台自动启动器
======================================================================
🕐 当前时间: 2025-12-26 09:45:30 ET

----------------------------------------------------------------------
📊 系统状态:
  🟢 子系统状态: 运行中
  📡 数据采集器: 工作中
  📺 监控面板: 运行中
  🔴 Redis: 运行中

----------------------------------------------------------------------
📈 市场状态:
  🟢 市场状态: 交易中
  ✅ 子系统: 正常工作

----------------------------------------------------------------------
🎮 控制命令:
  Ctrl+C: 手动关闭启动器和所有子系统
  启动器持续监控交易时间...
======================================================================
```

**行为说明**:
- **交易日前**: 显示开盘倒计时和启动倒计时
- **开盘前3分钟**: 自动唤醒并启动所有子系统
- **交易时间**: 监控子系统运行状态
- **收盘后2分钟**: 自动关闭所有子系统的终端窗口
- **非交易日**: 保持监控，等待下一交易日

### 方法二：一次性启动模式

**文件**: `start_system.bat` (备用)

**操作**:
1. 运行 `python start_system.py` (不使用批处理)
2. 系统会自动启动所有组件
3. 执行完成后窗口自动关闭

**特点**:
- 手动启动，不自动调度
- 适合测试和调试

## 📋 技术实现

### 调度逻辑
- **开盘前3分钟**: 检测到时间到达，自动启动Redis、监控面板、数据采集器
- **收盘后2分钟**: 检测到时间到达，自动终止所有子系统的进程及其终端窗口
- **进程管理**: 使用psutil库智能终止进程树，包括cmd.exe终端窗口

### 状态显示
- **实时刷新**: 每30秒更新一次显示
- **市场感知**: 根据交易日历显示相应的倒计时
- **状态监控**: 显示各个子系统的运行状态

### 错误处理
- **进程清理**: 启动前自动清理可能的残留进程
- **优雅关闭**: 支持Ctrl+C信号优雅关闭
- **日志记录**: 所有操作都有详细日志

## 🛠️ 故障排除

### 问题：启动器启动失败
**检查**:
- Python是否正确安装
- 系统PATH是否包含Python
- 相关依赖是否安装

### 问题：子系统启动失败
**检查**:
- 端口是否被占用 (Redis:6379, 监控面板:8050)
- 相关文件是否存在
- 日志中的错误信息

### 问题：终端窗口未关闭
**检查**:
- psutil库是否安装
- 进程是否有足够权限被终止
- 是否有其他进程占用了相关端口

### 问题：Redis 实盘数据未保存
**检查**:
- 确认 `RedisClient.store_breadth_data` 已写入 `trading_date:{YYYY-MM-DD}` 有序集合（已在代码第 425‑426 行实现）。
- 使用 `redis-cli` 查看键 `trading_dates` 与 `trading_date:2026-01-14` 是否存在。
- 若缺失，重新启动监控服务或手动运行数据采集脚本以写入历史数据。
- 确认 `settings.REDIS_HOST`、`REDIS_PORT` 正确指向运行中的 Redis 实例。
